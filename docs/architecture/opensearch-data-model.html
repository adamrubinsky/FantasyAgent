OpenSearch Vector Database - Data Model Design

Index: player_performance

Stores historical performance data with embeddings for semantic search on player patterns.

Field Name Type Description Example
player_id keyword Unique player identifier "CHM2023"
performance_embedding dense_vector Vector representation of player's performance pattern [0.123, -0.456, ...]
performance_narrative text Natural language description for embedding "Consistent RB1 with strong goal-line usage..."
boom_bust_score float Volatility metric (0-1) 0.73
Embedding Strategy: Combine statistical patterns with narrative descriptions to create rich semantic representations. Example: "High-floor PPR back with 15+ touches per game, excels in negative game scripts"

Index: draft_history

Tracks draft patterns and value identification with context-aware embeddings.

Field Name Type Description Example
draft_value_score float How much value above/below ADP +15.3
draft_context object League settings and roster state at pick {"format": "PPR", "roster_needs": ["RB", "WR"]}
outcome_rank integer End of season positional rank 5

Index: player_insights

Rich analytical insights with embeddings for nuanced player understanding.

Field Name Type Description Example
injury_history object Detailed injury tracking {"2024": ["hamstring", "ankle"], "games_missed": 3}
usage_trends object Target share, snap %, red zone usage {"target_share": 0.28, "red_zone_targets": 15}
breakout_indicators object Predictive metrics for improvement {"age": 23, "opportunity_score": 0.85}
